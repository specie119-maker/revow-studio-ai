[최종 지시문: 'Restored Light' 웹 서비스 개발 프로젝트]

안티, 지금부터 우리 서비스의 마이그레이션과 신규 기능 구현을 동시에 진행할 거야. 아래 내용을 바탕으로 사용자 중심의 고퀄리티 복원 웹사이트를 완성해 줘.
1. 프로젝트 목표 및 철학
핵심: 사용자가 올린 소중한 사진을 고퀄리티로 복원하고, 선택한 의상/배경 스타일을 입혀 새로운 추억으로 재탄생시킨다.
UX 원칙: 디자인은 간결하게, 'Before/After 비교 슬라이더'를 통해 복원 효과를 극대화한다.
2. 사용자 인터페이스(UI) 구성 (제공된 디자인 적용)
디자인 시스템: 내가 전달한 컬러 팔레트와 폰트 가이드를 엄격히 준수할 것.
레이아웃: 모바일 중심 반응형(Responsive) 설계.
헤드라인: "함께한 시간은 충분하지만, 정작 그날의 사진이 없으신가요?"
Before/After 슬라이더: 페이지 중앙에 배치하여 사용자가 즉각적으로 변화를 체감하게 할 것.
옵션 선택 영역: 국가별 전통 예복 및 현대 스타일, 배경 옵션을 아이콘과 함께 직관적으로 배치.
3. 핵심 기능 구현 로직
2단계 프로세스 도입:
1단계 (미리보기): 사용자가 옵션(의상/배경) 버튼을 클릭할 때마다 API 호출 없이 미리 준비된 샘플 이미지로 스타일을 즉시 시각화해 줘. (비용 절감 및 체감 속도 향상)
2단계 (AI 복원/생성): 최종 [복원하기] 버튼 클릭 시, 사용자의 사진과 선택된 옵션을 조합하여 Gemini API(Nano Banana 2 등)를 호출해 고퀄리티 이미지를 생성할 것.
감성 로딩 시스템: API 통신 중 버튼 텍스트가 2.5초 간격으로 롤링되며 나타나게 할 것.
"소중한 추억이 조금씩 선명해집니다..."
"가장 아름다웠던 순간을 그리는 중입니다..."
"당신의 기억 속 색을 찾는 중입니다..."
4. 기술 및 운영 요구사항
이미지 처리: assets 폴더에 이미지를 저장하지 말고, 프롬프트 템플릿 조합 방식을 사용하여 동적으로 생성할 것.
비용 관리 (중요):
테스트 단계에서는 Mock 데이터를 활용할 것.
실제 API 호출 시, 사용자별 무료 횟수 제한 로직을 구현할 것.
사용자가 본인의 API 키를 입력하면 요금이 분담되는 BYOK(Bring Your Own Key) 모드를 대비할 것.
저작권: 모든 이미지는 API를 통한 생성형 AI 방식으로만 처리하여 저작권 이슈를 원천 차단할 것.
작업 방향:
위의 지시사항을 바탕으로, 우선 [레이아웃 스켈레톤 구현] → [스타일 선택 미리보기 로직] → [API 연동 및 결과물 표출] 순서로 단계적으로 진행해 줘. 각 단계가 완료될 때마다 브라우저에서 확인할 수 있게 해 줘. 바로 시작할 수 있지?
