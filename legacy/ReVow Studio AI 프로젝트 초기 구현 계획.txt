
ReVow Studio AI 프로젝트 초기 구현 계획입니다.

1. 앱 주제 및 핵심 기능

앱 이름:
ReVow Studio AI

브랜드 메시지:
Love, Once Again.

한국어 메시지:
기다려온 순간을 AI가 사진으로 완성합니다.

영문 설명:
AI Wedding & Memory Portrait Studio

서비스 성격:
사용자가 본인 또는 가족 사진을 업로드하면 AI가 웨딩사진, 기념사진, 추억복원 이미지, 특별한 순간의 포트레이트를 생성해주는 모바일 웹앱(PWA)입니다.

초기 버전은 실제 AI 생성 API 연결 전까지는 프론트엔드 중심으로 제작하고, 업로드 → 스타일 선택 → 생성 대기 → 결과 화면까지 UX 흐름을 완성합니다.

핵심 기능:
- 모바일 중심 홈 화면
- Before / After 예시 화면
- 사진 업로드 화면
- 생성하고 싶은 스타일 선택
- AI 생성 대기 화면
- 결과 이미지 화면
- 저장 / 공유 버튼 UI
- 개인정보 보호 안내
- PWA 설치 가능 구조


2. 기술 스택

우선은 단순하고 배포 쉬운 구조로 진행합니다.

추천 기술:
Vanilla JS + HTML + CSS

이유:
- 초기 MVP 제작이 빠름
- GitHub Pages / Firebase Hosting / Netlify 배포 쉬움
- 추후 React 또는 앱 패키징으로 전환 가능
- 현재 디자인 확인과 모바일 테스트에 적합

단, 프로젝트가 커지면 이후 React + Vite로 전환 가능하도록 파일 구조를 깔끔하게 분리합니다.

13. 현재 단계에서 실제 AI API는 연결하지 않음

다만 추후 AI API 연결이 쉽도록 app.js 안에 다음 함수 구조를 만들어주세요.

selectStyle()
startGeneration()
showResult()
saveImage()
shareImage()

14. 개발 원칙

- 모바일 화면 우선
- 한 파일에 모든 코드를 몰아넣지 말고 css/js 분리
- 이미지 경로는 assets/images 또는 assets/samples 사용
- PWA 관련 파일은 루트에 배치
- 실제 배포 전까지 결제 기능은 구현하지 않음
- 로그인 기능은 1차 MVP에서 제외
- 개인정보 보호 문구는 반드시 표시
- 고객 사진은 공개하지 않는다는 메시지를 명확히 표시


15. 최종 목표

이번 1차 작업 목표는 실제 AI 생성이 아니라
“앱처럼 보이고 작동하는 ReVow Studio AI 모바일 웹앱 프로토타입”입니다.

완료 후 확인할 것:
- 모바일에서 화면 깨짐 없는지
- 홈 화면 메시지가 명확한지
- 사진 생성 앱이라는 것이 3초 안에 이해되는지
- PWA 설치 가능 구조가 있는지
- 추후 네이티브 앱 웹뷰 패키징이 가능한 구조인지
